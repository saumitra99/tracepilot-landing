Built for developers who live in their tools.

Tracepilot reads each tool's local transcript store. No proxy. No CA cert. No env vars. Just install and open the dashboard.

◇

Local-first by default

SQLite at ~/.tracepilot/traces.db. Dashboard at localhost:4321. Your data never leaves your machine.

◈

Multi-tool, one view

Claude Code JSONL, Cursor SQLite, Copilot logs, and Antigravity — unified into one timeline of tasks.

◉

Cache-aware pricing

Anthropic ephemeral 5m/1h cache writes and reads priced correctly. Per-task token and dollar totals you can trust.

◊

Zero setup mode

Default mode is pure file-watching. No MITM proxy, no certificate dance. Just run tracepilot start.

◍

Optional live capture

Want raw network traces too? tracepilot proxy enable spins up an opt-in HTTPS MITM, fully reversible.

◐

Single binary

One Rust binary per platform. No Electron. No Tauri. The UI is plain HTTP served to your browser of choice.

Built for solos & teams

One trace stream. Every lens you need.

Whether you're a solo dev getting sharper, or a team that needs visibility across engineers, leads, and budget — same data, different views.

◆

For individual developers

My Coaching

Spot your own patterns before anyone else does. Self-coaching, no team required.

  • Rework rate. How often your follow-up prompt says "fix / broken / wrong".
  • Cache hit ratio. How much context you're paying to re-send each turn.
  • First-prompt success. Tasks closed in one human turn vs many.
  • Hour-of-day cost curve. When you spend most — and whether tired hours are expensive hours.
  • Top expensive tasks. Click into any task to see the prompt, the tool calls, and the cost trail.
  • Model fit. Opus on a Read-only task is overkill — see the cheaper equivalent.

◇

For tech leads

Team Quality

Coach the team with data, not vibes. Spot drift before retros, not after.

  • Per-dev quality table. Spend · tasks · rework rate · cache hit · anchor coverage · top tool · skill use — at a glance for everyone you lead.
  • Top expensive tasks, team-wide. Postmortem queue. Click any row → full prompt + tool trail.
  • Repeated knowledge gaps. Three devs asking the same question = a missing doc. Tracepilot flags it.
  • Skill & subagent adoption. Who's leveraging parallel agents, code-reviewer, and domain skills — and who isn't.
  • Branch concurrency. Daily distinct anchors per dev — context-switching tax made visible.
  • Onboarding curve. New joiner's tools-used-per-week growing over time.
◈

For managers

Org & Budget · team scope

Track the team in aggregate, then drill into any developer or any ticket without leaving the page.

  • Spend per dev, per week. Burndown chart. One click on a dev → that person's My Coaching view (read-only).
  • Spend per Jira / Linear ticket. Cost attached to anchors. Cross with story points for $/point.
  • End-of-month forecast. 7-day rolling avg × days remaining. Know before finance asks.
  • Model-mix simulator. "Shift 30% Opus → Sonnet, save $143/mo." Run before you pull the lever.
  • Anomaly alerts. Dev's daily cost > 3σ above baseline → Slack ping. Routed to you, cc'd to the owner on spikes.
  • Off-hours spend. % cost outside 9–6 — burnout proxy you can act on.
◉

For owners

Org & Budget · full scope

Defend the AI budget to the board. Every dollar attributed, every paid tool justified.

  • Org-wide burndown. All teams, all repos, all paid tools. One number, drill all the way down.
  • Cache-saved $ headline. "Caching paid for Cursor seats this week." Concrete savings line you can quote.
  • Plan utilization. Live ceiling vs burn per paid tool (Claude Code Max, Cursor, ChatGPT Team, Gemini).
  • Tool subscription justification. Spend per app vs seats. Auto-suggest keep / downgrade / drop.
  • Per-product attribution. Group spend by repo / topic / business line. Know which product the AI bill is buying.
  • $/PR proxy. When dev anchors match merged PRs, surface cost-per-shipped-PR. The closest thing to AI ROI.

Privacy isn't a feature. It's the architecture.

Tracepilot is local. There is no Tracepilot cloud. There is no analytics endpoint. The daemon never opens a socket to anything we own.

  • × No keystroke logging. Ever.
  • × No screen capture. Ever.
  • × Clipboard text hashed before storage. Raw text never persisted.
  • ✓ Prompts only read from sources you already have locally.
  • ✓ Everything under ~/.tracepilot/.
  • ✓ tracepilot uninstall removes it all.
